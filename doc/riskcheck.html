Pre-Trade Risk Check

Pre-Trade Risk Check verifies order requests generated by strategies before they are passed to the Order Sender where order message is generated and sent to market. There is one set of global limits (maximal & minimal price, maximal & minimal size) for all instruments. Pre-Trade Risk Check module setting can be set and changed by set of methods available in Magmio API. The methods are described in Doxygen documentation and their usage is shown in API example code.

Module modes

Pre-Trade Risk Check can be enabled or disabled as necessary. When disabled, no order verification is made and all order requests received at the module input are sent to the output. When enabled, every order request is checked. When the order price and size fits the configured limits (including equality), order request is passed and sent to the order sender. When the order price or size don’t fit the configured limits, order request is dropped. In the default setting, the module is disabled.

Check algorithm

When Pre-Trade Risk Check is enabled, price and size of the order request are checked against configured minimum and maximum limits. These limits can be set and read via methods available in Magmio API.

For quote markets, both ask and bid sides are checked. Bid size or bid price is allowed to be zero, even though the minimum limits are set to non-zero value.

Getting the result

Pre-Trade Risk Check result of individual orders can be monitored by reading strategy stream output. The module also contains counter of discarded orders. Every time an order is discarded in Pre-Trade Risk Check module, counter is incremented. Counter can be read and reset via appropriate methods in Magmio API.

User can also subscribe for EVT_PRE_TRADE message from the status channel, which indicates that the pre-trade unit has eliminated order with invalid price/size.
